Translation Proxy Server

by Cesar Gonzalez

DESCRIPTION

This web application implements a transparent proxy interface to the World Wide Lexicon (WWL) API, which is an online translation memory. The idea behind the proxy is to give the webmaster a simple interface to implement translations of their website into any desired language, simply by pointing a language code subdomain (eg. http://es.mywebsite.com/) at a server where this proxy code is running. There are several distinct advantages to using this method to translate your website:
  • Simplicity. Install the proxy (or use ours), change one A-record, tweak some setting, and presto!
  • Standard URL structure. For any page on your site, you know exactly how to access that page in any available language.
  • SEO & indexability. Many translation tools are javascript-based, which means the search engines will not index your content in other languages. Using the WWL proxy, you have the option of making your content searchable in other languages.
  • Supports any combination of machine translation, community generated translations, and professional translation agencies.
  • Works with any HTML compatible web server. No software to install on your original website.
  • Fast. In memory caching (memcached) for fast page delivery.

QUICK START

Installing and using the translation proxy server is easy. Just perform the following tasks:

  1. Download the PHP code
  2. Install the code on a web server that has PHP installed
  3. Map several domains to this server, such as fr.yoursite.com
  4. Go to fr.yoursite.com to view it in French.
  5. Repeat as needed for other languages, where __.yoursite.com is a 2 or 3 letter ISO language code

REQUIREMENTS

  • PHP 5 (Tested on PHP 5.1.6)
  • cURL (libcurl/7.15.5)
  • Memcache *recommended* (v 2.2.3)

INSTALLATION (PROXY SERVER)

  1. Unzip the archive and FTP all the files into the base web directory on your server.
  2. Open includes/config.php in your favorite editor and:
  3. Define your server IP, home directory path, and log path
  4. If you have memcache, define the address, port, etc.
  5. Define which domains to translate in the whitelist / blacklist section
  6. Create /logs and /cache directories (if enabled) and set them editable by the server.

SETUP (WEBSITE TO TRANSLATE)

  1. Go to your domain registrar and create an A-record pointing to the IP address of the Proxy:
  2. Set up one A-record using a wildcard (eg. *.mywebsite.com)
  3. OR set up multiple A-records, each with a 2-letter language code (ISO 639-1) as a subdomain
That's it! Now you should be able to access http://**.mywebsite.com/ and see it another language

You can fine-tune setting using META tags:

Usage: <meta name="{setting_name}" content="{setting_value}" />
  • allow_machine (y/n)
  • allow_anonymous (y/n)
  • allow_edit (y/n)
  • require_professional (y/n)
  • languages (string, comma-separated list)
  • machine_translation_languages (string, comma-separated list)
  • community_translation_languages (string, comma-separated list)
  • professional_translation_languages (string, comma-separated list)
  • lsp (string)
  • lspusername (string)
  • lsppw (string -- more secure way?)
Examples:

<meta name="languages" content="es, fr, br, ar, de" />
<meta name="allow_edit" content="n" />
You can fine-tune whether a give section of the page can be translated / edited using CSS:

Usage: <{tag} class="{setting}"> </{tag}>
  • allow_edit
  • disallow_edit
Examples:

<div class="header disallow_edit"> . . . . &;tl/div>
<p id="callout" class="highlight allow_edit"> . . . . </div>
